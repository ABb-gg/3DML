L'évolution des technologies 3D sur le Web

De 3DML aux frameworks modernes

Introduction

L'histoire de la 3D sur le Web est marquée par des technologies pionnières qui ont ouvert la voie aux expériences immersives que nous connaissons aujourd'hui.

Parmi ces technologies, 3DML (3D Markup Language) occupe une place particulière comme l'un des premiers langages dédiés à la création de mondes 3D directement dans le navigateur.

Saviez-vous ? 3DML a été développé en 1998 par Flatland Online, bien avant que les navigateurs ne supportent nativement la 3D.

Qu'est-ce que 3DML ?

Définition

3DML (3D Markup Language) était un langage de balisage XML-like conçu pour créer des mondes virtuels 3D navigables dans un navigateur web.

Caractéristiques

  • Syntaxe similaire à HTML
  • Description déclarative des scènes 3D
  • Optimisé pour le web avec des fichiers légers
  • Support des textures, animations et interactions
  • Nécessitait le plugin Flatland Rover

Exemple de code 3DML

<world name="monde3d">
    <block name="sol" position="0,0,0" size="10,1,10" texture="grass.jpg"/>
    <block name="mur1" position="0,0,5" size="10,5,1" texture="brick.jpg"/>
    <object name="arbre" position="3,0,3" model="tree.3dml"/>
    <spotlight name="lumière" position="5,10,5" target="0,0,0"/>
</world>

Cet exemple montre comment une simple scène 3D pouvait être décrite avec 3DML, avec des éléments comme des blocs, des objets et des lumières.

Flatland Rover - Le navigateur 3D

Qu'était Flatland Rover ?

Flatland Rover était un navigateur 3D spécialisé qui permettait d'afficher des mondes créés en 3DML. Il fonctionnait comme un plugin pour les navigateurs web traditionnels.

Contrairement aux solutions modernes qui utilisent WebGL, Flatland Rover avait son propre moteur de rendu 3D.

Fonctionnalités clés :

  • Navigation intuitive dans les mondes 3D
  • Support des interactions utilisateur
  • Affichage en temps réel
  • Optimisation pour les connexions lentes

Comparaison : 3DML vs Frameworks modernes

Caractéristique 3DML (2000) Frameworks modernes
Syntaxe XML/HTML-like JavaScript (Three.js, Babylon.js) ou HTML-like (A-Frame)
Rendu Plugin propriétaire (Flatland Rover) WebGL/WebGPU natif dans le navigateur
Performance Limitée par la technologie de l'époque Haute performance, matériel accéléré
Compatibilité Nécessitait l'installation d'un plugin Fonctionne sur tous les navigateurs modernes
Communauté Petite communauté, projet fermé Grandes communautés open-source
Fonctionnalités Basiques : formes, textures, lumières Avancées : shaders, physique, VR/AR, etc.

Exemple A-Frame

<a-scene>
    <a-box position="-1 0.5 -3" rotation="0 45 0" color="#4CC3D9"></a-box>
    <a-sphere position="0 1.25 -5" radius="1.25" color="#EF2D5E"></a-sphere>
    <a-cylinder position="1 0.75 -3" radius="0.5" height="1.5" color="#FFC65D"></a-cylinder>
    <a-plane position="0 0 -4" rotation="-90 0 0" width="4" height="4" color="#7BC8A4"></a-plane>
    <a-sky color="#ECECEC"></a-sky>
</a-scene>

Notez la similarité conceptuelle avec 3DML, mais avec une implémentation moderne.

Exemple Three.js

// Création d'une scène
const scene = new THREE.Scene();
const camera = new THREE.PerspectiveCamera(75, window.innerWidth/window.innerHeight, 0.1, 1000);
const renderer = new THREE.WebGLRenderer();

// Création d'un cube
const geometry = new THREE.BoxGeometry();
const material = new THREE.MeshBasicMaterial({color: 0x00ff00});
const cube = new THREE.Mesh(geometry, material);
scene.add(cube);

// Animation
function animate() {
    requestAnimationFrame(animate);
    cube.rotation.x += 0.01;
    cube.rotation.y += 0.01;
    renderer.render(scene, camera);
}

Approche plus programmatique que déclarative.

Impact et héritage de 3DML

Contributions importantes

  • Pionnier du 3D web : A démontré que le web pouvait être un support pour des mondes 3D
  • Approche déclarative : A inspiré des frameworks comme A-Frame
  • Optimisation réseau : Solutions pour des mondes 3D sur connexions lentes
  • Navigation intuitive : Modèles de navigation qui influencent encore les interfaces 3D

Leçons apprises

  • Les plugins propriétaires étaient une impasse pour le web ouvert
  • La standardisation (WebGL) était nécessaire pour une adoption large
  • L'approche déclarative (comme 3DML) reste valable pour certains cas d'usage
  • La performance et l'accessibilité sont cruciales pour l'adoption

État actuel du 3D web

Aujourd'hui, le 3D sur le web repose sur des standards ouverts comme WebGL et bientôt WebGPU, avec des frameworks matures comme Three.js, Babylon.js et A-Frame.

Les technologies comme WebXR permettent même des expériences de réalité virtuelle et augmentée directement dans le navigateur, réalisant enfin la vision que des pionniers comme 3DML avaient imaginée il y a plus de 20 ans.

Conclusion

3DML a été une technologie pionnière qui a ouvert la voie au 3D sur le web, avec une approche innovante mais limitée par la nécessité d'un plugin propriétaire.

Les frameworks modernes comme Three.js, Babylon.js et A-Frame réalisent aujourd'hui cette vision avec des standards ouverts, des performances accrues et une adoption massive.

L'histoire de 3DML nous rappelle l'importance des standards ouverts et de l'évolution constante des technologies web.
